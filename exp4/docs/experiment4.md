# 实验四 — 虚拟机汇编代码输出与运行

> 来源：Mandrill 参考手册（docs/mandrill-v2-intro.pdf）节选并整理

## 实验目的与总体要求

- 实验四权重：30%。
- 从 `stdin` 读入 Mandrill 源代码，输出字节码到 `stdout`。
- 评测机上的虚拟机将从 `stdin` 读入该字节码并执行，输出结果到 `stdout`。
- 输入的源代码保证不存在词法和语法错误。
- 要保证源代码在给定输入的情况下不会发生死循环（程序正常终止）。

## 字节码与虚拟机模型约定

- 每条指令占 8 个字节。
- 程序计数器（PC）为 32 位，从 `0x00000000` 开始。
- 正常执行每条指令后 PC 自动加 8；跳转/分支等指令会修改 PC。
- 当 PC 恰好等于 `0xFFFFFFFF` 时，虚拟机退出并返回 0（通常在程序末尾生成 `jump 0xFFFFFFFF` 作为结束指令）。
- 虚拟机维护一个操作数栈（用于存放中间值与函数返回值），以及寄存器：PC 与栈指针（SP，指向第一个局部变量地址）。

## 功能性需求（总体）

- 实现字节码生成器：将 Mandrill 源程序翻译为符合上述约定的字节码流。
- 字节码由评测机虚拟机解释执行，需保证行为一致（包括 I/O、堆分配、函数调用等）。
- 正确处理局部/全局作用域、函数调用/返回、递归、动态内存分配与字符串（UTF-32 存储）。

## 指令集要点（编译器需要生成这些指令）

### 数据存取指令（入栈）
- `dload x`：全局数据加载，从数据区读取变量编号 `x` 的值并压入操作数栈。
- `dlload x`：局部数据加载，从局部数据区（相对于 SP 的偏移）读取编号 `x` 的值并压栈。
- `daload x`：绝对数据加载，从栈顶取地址，并从该内存地址读取数据压入栈（用于访问数组元素）。`x` 为无用填充。
- `dstore x`：数据存储指令，将立即数 `x`（或常量）压入操作数栈（即入栈立即数）。

### 数据写入指令（出栈）
- `dwrite x`：将操作数栈顶元素写入对应的存储位置（全局），并弹出。
- `dlwrite x`：将操作数栈顶元素写入对应的局部存储位置，并弹出。
- `dawrite x`：绝对数据写入，从栈顶取地址（顶）、写入值（次顶），将两者弹出并写入对应内存；`x` 为无用填充。

### 求值指令
- `eval x`：按操作码 `x` 执行运算或条件跳转。
  - 若 `x` 为二元算术/比较运算，则弹出两个操作数计算后将结果压栈。
  - 若 `x` 为条件跳转（文档中的条件跳转码），则从栈中取三个操作数：第 1（原栈顶）为条件结果；第 2 为 THEN 地址；第 3 为 ELSE 地址。如果条件非零，将 PC 设为 THEN 地址，否则设为 ELSE 地址。
  - 编译器在生成条件跳转时需预先计算 THEN/ELSE 的目标地址并将它们作为立即数通过 `dstore` 压入栈中。

### 跳转与控制流
- `jump x`：将 PC 直接设为 `x`。
- `jal x`：链接跳转（call）：记录当前栈寄存器和 PC，并根据栈顶元素申请内存，把栈寄存器设置为新的内存地址，然后将 PC 设为 `x`。
- `ret x`：返回指令，从虚拟机控制的存储区恢复原先的栈寄存器与 PC，恢复后继续执行。`x` 忽略。
- `nop 0`：无操作。

### 内存分配
- `malloc x`：从栈顶取出一个元素作为要分配的字节数，申请对应大小的内存并将起始地址压入操作数栈；`x` 为无用填充。
  - 对于字符串字面量，需生成 `malloc` 并逐字符复制的指令序列，同时保证字符串以 `0x00000000` 结尾（UTF-32）。

### 输入/输出指令
- 整数输入：`geti x`（操作数无用）——从 `stdin` 读入十进制整数并压入栈顶。
- 字符输入：`getc x`（操作数无用）——读入一个字符的 ASCII 值并压栈。
- 字符串输入：`gets x`（操作数无用）——读入连续非空白字符构成的字符串并将其地址压栈（用于 `read`）。
- 整数输出：`puti x`（操作数无用）——弹出栈顶元素并以十进制形式输出到 `stdout`。
- 字符输出：`putc x`（操作数无用）——弹出栈顶元素并转换为 UTF-8 字符输出到 `stdout`（若不合法则忽略）。
- 字符串输出：`puts x`（操作数无用）——弹出栈顶元素，若为合法内存地址则从该地址按 C 风格字符串（以 0 结尾）读取 UTF-32 内容并转换为 UTF-8 输出。

### 函数调用语义（编译器生成策略）
- 调用者将函数参数依次压入操作数栈（按声明顺序）。
- 被调用者一次性从操作数栈弹出参数并存入对应局部变量位置。
- 被调用者执行完毕后将返回值（若有）压入操作数栈；调用者随后从栈中取出返回值并用于后续运算。
- 编译器需为 `jal/ret` 生成正确的栈帧建立/恢复逻辑，并处理递归调用。

## EVAL 指令操作数（部分枚举，按文档）
- 加法：`0x00010001`
- 减法：`0x00010002`
- 乘法：`0x00010003`
- 整除：`0x00010004`
- 求余：`0x00010005`
- 大于：`0x00010006`
- 小于：`0x00010007`
- 大等于：`0x00010008`
- 小等于：`0x00010009`
- 等于：`0x0001000A`
- 不等于：`0x0001000B`
- 条件跳转：`0x0001000C`

> 编译器在生成条件跳转时，常见策略是：先计算并用 `dstore` 将 THEN/ELSE 目标地址压栈，再生成 `eval 0x0001000C`。

## 实现建议与注意事项

- 符号表设计：记录 `Name`、`Type`（INT 或 ARRAY_PTR）、`Scope`（GLOBAL/LOCAL/层级）、`Offset/Address`（栈帧偏移或全局地址/编号）。
- 变量编号/偏移规则需与 `dload/dstore/dwrite` 等指令编码一致（统一约定全局变量从 0 开始编号，局部用相对偏移）。
- 字符串处理：源文件为 UTF-8，需要在编译期或运行时将其转换为 UTF-32 存储，且每个字符占 4 字节，数组索引按 4 字节递增。
- 数组索引地址计算：Address = Base(a) + (i * 4)。
- 内存管理：实现基本的 `malloc`/`free` 语义（或由 VM 提供），并在 `a = read` / `a[] = read` 时正确释放旧内存并分配新内存。
- 跳转地址回填：由于每条指令固定宽度（8 字节），在一次遍历时可以计算出目标指令地址（指令序号 * 8 + 基址），或使用两遍生成法（先收集长度再回填）。
- 结束指令：在程序末尾生成 `jump 0xFFFFFFFF`，以保证 VM 退出。
